# Stage 15 Closed Testing 申请包

## 1. 目标
从 internal testing 过渡到 closed testing，验证更广泛设备与真实网络场景。

## 2. 申请材料清单
- [ ] 应用简介与目标用户
- [ ] 功能变更说明（相对 internal）
- [ ] 已知问题列表（OPEN-ISSUES）
- [ ] 测试重点与风险控制
- [ ] 隐私政策 URL 与数据安全声明

## 3. 建议测试规模
- 50~200 名测试者
- 覆盖 Android 10~14 不同机型

## 4. 通过门槛
- P0 = 0
- P1 < 3 且均有修复计划
- 关键路径成功率 >= 98%
